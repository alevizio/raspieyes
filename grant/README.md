# Madrone Artist Grant — "The Regulars"

Application materials for the Madrone Art Bar Artist Grant
(madroneartbar.com/madrone-artist-grant), submission window April 1 –
June 1, fall exhibition at 500 Divisadero St., San Francisco.

## Files in this directory

| File | Purpose |
|---|---|
| `artist_statement.md` | Long-form artistic framing. Use for the "statement" field on the application, or as press-release copy if selected. |
| `project_description.md` | ~250-word application-form project description. Drop directly into the form. |
| `artist_bio.md` | Fill-in-the-blank bio template with explicit guidance on word choices and what to avoid. |
| `budget.md` | Line-item budget totalling $1,500, formatted for the application. |

## Visual materials still to prepare

Text alone will not sell this piece — the judges need to *see* the
concept. Capture these before submission:

1. **20-second video** of the 2-eye prototype tracking a person walking
   past it. Landscape, quiet, no narration. One good clip is worth
   more than every page in this directory.
2. **3–5 night-lit process photos** of the 2-eye prototype. Tight
   crops, warm key light, shot so the eyes glow against a dark
   background. No visible cables, no laptop in frame.
3. **1 mockup composite** of the 12-eye frame hung on a Madrone
   interior wall with a patron standing in front of it and several
   eyes visibly rotated toward them. A rough Photoshop paste-over is
   fine — it does not need to be photoreal, it needs to communicate
   scale and placement.
4. **1 front-view drawing** of the finished 12-eye layout
   (~900 × 600 mm). Use `../frame_layout.json` as the source of truth
   for eye positions and sizes. This signals to the panel that the
   piece is designed, not improvised.

## Positioning principle — read this before editing anything

The Madrone panel is a mixed-discipline visual-art jury (ceramics,
photography, etc.). They are choosing art that lives in a bar, not
tech that happens to hang on a wall. Every single edit to these
documents should **lead with the feeling, not the technology**. If a
sentence would make sense printed on a gallery card, it belongs here.
If it would only make sense in a GitHub README, it does not.

Words to avoid: "Raspberry Pi", "Python", "software", "code", "SPI",
"HDMI", "algorithm", "AI", "computer vision", "machine learning",
"detection", "tracking" (in the technical sense).

Words to lean into: "watching", "attention", "presence", "gaze",
"witnesses", "gathered", "hand-built", "bespoke", "quiet",
"patient", "uncanny", "intimate".
